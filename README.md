🛡️ End-to-End Threat Simulation & Network Intrusion Detection Lab📖 Project OverviewThis lab demonstrates a comprehensive Red vs. Blue team simulation. I developed custom offensive scripts to breach a target environment and simultaneously configured defensive telemetry to analyze attack patterns at the packet level. This dual-perspective approach is designed to simulate real-world Threat Hunting and Incident Response workflows.🏗️ 1. Lab ArchitectureThe project was executed in a strictly isolated virtual environment to ensure safety and ethical compliance.Attacker Node: Kali Linux (10.0.2.15)Victim Node: Metasploitable 2 (10.0.2.3)Monitoring Tools: Snort 3 (NIDS), Tcpdump, Wireshark⏳ 2. Attack Lifecycle (Timeline)Following the Cyber Kill Chain methodology, the compromise was executed in the following stages:Reconnaissance: Mapped the attack surface using a custom Python-based socket scanner.Service Enumeration: Identified vulnerable instances of SSH (22), Telnet (23), and MySQL (3306).Exploitation: Executed a database breach to perform SQL Data Exfiltration, dumping the user credential table.Persistence: Established post-exploitation stability by creating privileged backdoor users and performing web defacement.Detection Analysis: Analyzed real-time network telemetry to identify malicious signatures.⚔️ 3. Featured Attack: SSH Brute ForceVector: Automated Credential Stuffing.Tooling: Metasploit Framework (auxiliary/scanner/ssh/ssh_login).Observation: Successfully bypassed weak authentication mechanisms due to default credential usage.🕵️‍♂️ 4. Detection Analysis (Blue Teaming)Using Tcpdump and Snort 3, I identified several Indicators of Compromise (IOCs) during the simulation:IOC CategoryTechnical ObservationSeverityNetwork TrafficAbnormal spikes in TCP/22 traffic (SSH) within a 5-second window.HighPacket AnalysisHigh frequency of TCP [P.] (Push) flags indicating automated data transfer.MediumForensicsRepeated SYN/ACK sequences followed by immediate resets (RST).High🧠 5. Lessons Learned & Security HardeningThis lab highlights critical defensive gaps common in enterprise environments:Credential Management: Default credentials remain a primary entry point. Remediation: Implementation of MFA and strict password complexity.Visibility: Attacks are often invisible without deep packet inspection (DPI). Remediation: Centralized logging and real-time IDS alerting.Privilege Escalation: Service-level misconfigurations allowed for lateral movement. Remediation: Strict adherence to the Principle of Least Privilege (PoLP).🛠️ Technology StackLanguages: Python (Socket Programming), BashOffensive: Kali Linux, Metasploit, NmapDefensive: Snort 3, Tcpdump, Wireshark
